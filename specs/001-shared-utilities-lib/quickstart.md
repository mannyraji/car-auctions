# Quickstart: `@car-auctions/shared`

Generated by `/speckit.plan` | 2026-04-06

## Prerequisites

- Node.js 20+
- npm 10+ (ships with Node 20)
- Repository cloned; you are on branch `001-shared-utilities-lib`

---

## 1 — Bootstrap the Monorepo (first-time only)

The monorepo root does not yet have a `package.json`. This feature creates it.

```bash
# From the repo root — after this feature is implemented:
npm ci
```

`npm ci` reads the root `package.json` with `"workspaces": ["packages/*"]`, installs all workspace packages and hoists shared `devDependencies` to `node_modules/`.

---

## 2 — Build the Shared Package

```bash
npm run build --workspace packages/shared
# or from repo root:
npm run --workspaces --if-present build
```

Build output lands in `packages/shared/dist/`. TypeScript declaration files (`.d.ts`) and source maps are included.

---

## 3 — Run Tests

```bash
npm run test --workspace packages/shared
# or from repo root:
npm run --workspaces --if-present test
```

Tests run with Vitest. The `MemoryVinCache` is used in unit tests — no SQLite database is created in CI.

---

## 4 — Import in a Sibling Package

Declare the dependency in the consuming package's `package.json`:

```json
{
  "dependencies": {
    "@car-auctions/shared": "*"
  }
}
```

Then import:

```typescript
import {
  AuctionListing,
  normalizeCopart,
  decodeVin,
  BrowserPool,
  PriorityQueue,
  initTracing,
  withSpan,
  ScraperError,
} from '@car-auctions/shared';
```

> **Note**: Always import from `@car-auctions/shared` (the package root). Never import from sub-paths such as `@car-auctions/shared/dist/normalizer.js`.

## 5 — Usage Examples

### Normalize a Copart Listing

```typescript
import { normalizeCopart, CopartRawListing } from '@car-auctions/shared';

const raw: CopartRawListing = {
  lotNumberStr: '12345678',
  vn: '1HGCM82633A004352',
  mkn: 'Honda', lnn: 'Accord', yn: 2003,
  clr: 'Silver', dd: 'Front End', sdd: null,
  hk: true, ln: 'Lot 1', orr: 145_000, obd: 'Actual Miles',
  tmtp: 'SV', htrf: false,
  cd: 1200, bnp: null, sed: '2026-04-10T14:00:00Z',
  syn: 'Los Angeles', st: 'CA', pc: '90001',
};

const listing = normalizeCopart(raw);
console.log(listing.source);      // 'copart'
console.log(listing.title_type);  // 'Salvage' (mapped from 'SV')
console.log(listing.has_keys);    // true
```

### Decode a VIN (with in-memory cache for dev)

```typescript
import { decodeVin, validateVin, MemoryVinCache } from '@car-auctions/shared';

const vin = '1HGCM82633A004352';

if (!validateVin(vin)) {
  throw new Error('Invalid VIN');
}

const cache = new MemoryVinCache();
const response = await decodeVin(vin, cache);

if (response.success) {
  console.log(response.data?.make);   // 'Honda'
  console.log(response.cached);       // false on first call, true on second
}
```

### Decode a VIN (with SQLite cache for production)

```typescript
import { decodeVin, SqliteVinCache } from '@car-auctions/shared';
import path from 'path';

const cache = new SqliteVinCache({ dbPath: path.join('data', 'vin-cache.sqlite') });
const response = await decodeVin('1HGCM82633A004352', cache);
```

### Enqueue Scrape Requests with Priority

```typescript
import { PriorityQueue } from '@car-auctions/shared';

const queue = new PriorityQueue({ requestsPerInterval: 1, intervalMs: 3_000 });

// Critical: watchlist lot about to close — bypasses queue, respects rate limit
const result = await queue.enqueue(
  () => scrapeWatchedLot(lotId),
  'critical'
);

// Background: cache warm-up — will wait up to 30s before executing
queue.enqueue(
  () => warmSearchCache(query),
  'background'
);
```

### Browser Pool — Shared Context Management

```typescript
import { BrowserPool } from '@car-auctions/shared';

const pool = new BrowserPool({
  stealth: true,
  headless: true,
  maxContexts: 3,   // NFR-004: default max
  timeoutMs: 30_000,
});

const context = await pool.acquire();
try {
  const page = await context.newPage();
  await page.goto('https://example.com', { timeout: 30_000 });
  // ...
} finally {
  pool.release(context);   // returns context to pool
}

// When the process exits:
await pool.shutdown();   // idempotent, reference-counted
```

## 7 — Adding a New Package to the Monorepo

After this feature is merged, new workspace packages are added by:

1. Creating `packages/<pkg-name>/package.json` with `"name": "@car-auctions/<pkg-name>"`
2. Creating `packages/<pkg-name>/tsconfig.json` extending `../../tsconfig.base.json`
3. Adding `"@car-auctions/shared": "*"` to dependencies if the package needs shared types
4. Running `npm install` at the repo root to link the workspace
5. Adding `build`, `lint`, and `test` scripts — CI will pick them up automatically

---

## 8 — Linting and Formatting

```bash
# Lint all packages
npm run --workspaces --if-present lint

# Format (from root)
npx prettier --write .
```

ESLint flat config is defined in `eslint.config.mjs` at the repo root and applies to all `packages/*/src/**/*.ts` files.

---

## 9 — Data Directory

The `data/` directory inside each package is **gitignored** and created at runtime:

```
packages/shared/data/vin-cache.sqlite   ← created by SqliteVinCache on first use
```

Never commit files under `data/`. The SQLite databases are runtime state.
